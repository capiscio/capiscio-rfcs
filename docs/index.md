# CapiscIO RFCs

Request for Comments (RFCs) are the formal process for proposing and documenting significant changes to CapiscIO protocols and standards.

## Active RFCs

| RFC | Title | Status |
|-----|-------|--------|
| [RFC-001](001-agcp.md) | Agent Governance Control Plane (AGCP) | ✅ Approved |
| [RFC-002](002-trust-badge.md) | Trust Badge Specification | ✅ Approved |

## RFC Process

1. **Draft** — Initial proposal under discussion
2. **Review** — Formal review period (minimum 2 weeks)
3. **Approved** — Accepted for implementation
4. **Implemented** — Fully implemented in CapiscIO
5. **Superseded** — Replaced by a newer RFC
